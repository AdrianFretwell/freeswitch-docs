## transfer_on_fail

string Allows you to transfer call flow when a called party can not be reached for specific reasons (unallocated\_number, etc).

![(attenzione)](/images/icons/emoticons/warning.svg) you must also set [failure\_causes](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#failure_causes)

Argument syntax: `hangupcauses destination dialplan context`

**Usage**

```xml
<action application="set" data="failure_causes=UNALLOCATED_NUMBER"/>
<action application="set" data="transfer_on_fail=UNALLOCATED_NUMBER"/>
```

```xml
<action application="set" data="transfer_on_fail=1"/>
```
